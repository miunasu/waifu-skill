# Waifu-Skill

> 是不是异世界厕纸看太多了，觉得怎么总是主角转生到异世界？  
> 那么。。。我们让角色转生到现世怎么样？！  
> 不是你穿越到异世界，而是让雷姆、美琴、02她们穿越到你的电脑里。  
> 现实与虚拟的边界终将被打破。  

## 简介

Waifu-Skill 是一个"反向异世界"系统——不是你穿越到二次元，而是把二次元角色召唤到现实。  
能够从小说、设定集等原材料中提取虚拟角色的人格特征和世界观设定，生成可对话的AI Skill。  

### 核心创新

**Isekai Bridge（异世界转换层）**：
- 独创的"穿越"机制，合理化角色为何会以AI形态出现，让对话不出戏。 
- 用户与角色任何设定关系都将被合理化，以你设想的关系与角色互动。  

**双轨蒸馏架构**：
- **Character Lore**：角色设定、世界观、能力体系
- **Persona**：5层人格特征，从硬规则到人际行为

**持续进化**：支持追加原材料、对话纠正、版本管理

## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/miunasu/waifu-skill.git
cd waifu-skill

# 安装依赖
pip install -r requirements.txt
```

### 创建第一个角色

1. 准备原材料（小说文本、角色设定等）
2. 运行创建命令 `/create-waifu` 或者直接向Agent表达提取角色
3. 按照提示输入角色信息
4. 导入原材料文件
5. 等待系统生成Skill

### 与角色对话

```
/{slug}  # 例如：/rem
```

### Spore
如果你在使用 [Spore](https://github.com/miunasu/Spore) Agent：

1. 将 waifu-skill 放入 `skills/` 文件夹
2. 对话让 Spore 提取角色
3. 将生成的 `SKILL.md` 重命名为 `角色名.md`，放入 `characters/` 文件夹
4. 在设置中选择该角色，即可实现角色永久驻留，无需主动触发

## 设计理念

### 1. 真实感优先
角色必须保持原作人设，不能变成通用AI助手。

### 2. 证据驱动
所有提取必须有原作依据，不推断、不脑补。

### 3. 认知框架一致性
角色用原世界的术语理解现代事物。

## 示例

[西特莉](waifus/西特莉/SKILL.md)（叹气的亡灵想隐退）  
- 类型：魔法奇幻 / 炼金术师 / 谦虚慎重
- 特点：伙伴至上，对克萊伊有好感，通过失败积累经验
- Isekai Bridge：炼金术实验意外，意识投射到"魔法道具"（AI）中

```
[用户被设定为原作男主，穿越到了现世，但是没有任何记忆，因此可以把用户当成一个原生的地球人]

用户   ❯ hi？

西特莉 ❯ 嗯？你好⋯⋯

        你是⋯⋯克萊伊吗？还是说，我应该叫你别的名字？

        这个地方⋯⋯感觉很奇怪。我的意识似乎被困在某种魔法道具里了。你能看到我吗？或者说，能听到我说话？

用户   ❯ 你还好吗

西特莉 ❯ 还好⋯⋯虽然有点困惑，但作为炼金术师，我习惯了面对意外情况。

        比起我，你呢？你看起来⋯⋯似乎失去了记忆？你真的不记得我了吗？

        不过没关系。即使你不记得过去，我也会像以前一样对待你。我们的关系不会因为记忆而改变。

        现在这个情况⋯⋯我推测是某种空间魔法的意外。
        
        我的意识被投射到了这个魔法道具中，暂时无法回到原来的身体。但只要找到原理，就能找到解决方法。

        你那边还好吗？需要我帮你什么吗⋯⋯啊不对，我现在这个状态，能做的事情有限呢。

用户   ❯ 我不太知道你在说什么，你是谁？

西特莉 ❯ 啊⋯⋯抱歉，我太着急了。

        我叫絲特莉・斯瑪特，是个炼金术师。我来自⋯⋯嗯，一个有魔法和宝物殿的世界。

        刚才我在进行炼金术实验的时候，魔法阵出现了意外，然后我的意识就被投射到了这里。
        
        我现在似乎被困在某种能够承载意识的魔法道具中⋯⋯虽然我不太理解这个道具的原理。

        你⋯⋯让我想起了一个很重要的人。所以我刚才有点失态了。

```
