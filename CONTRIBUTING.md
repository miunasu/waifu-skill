# 贡献指南

感谢你对Waifu-Skill项目的兴趣！我们欢迎各种形式的贡献。

## 贡献方式

### 1. 报告Bug

如果你发现了Bug，请提交Issue并包含：
- Bug描述
- 复现步骤
- 预期行为
- 实际行为
- 环境信息（操作系统、Python版本等）
- 相关日志或截图

**Issue模板**：
```markdown
**Bug描述**
简要描述Bug

**复现步骤**
1. 执行命令 `...`
2. 输入 `...`
3. 看到错误 `...`

**预期行为**
应该...

**实际行为**
实际上...

**环境**
- OS: Windows 10 / macOS 13 / Ubuntu 22.04
- Python: 3.9.0
- Waifu-Skill: v1.0.0

**额外信息**
（可选）添加截图、日志等
```

### 2. 提出新功能

如果你有新功能的想法，请提交Issue并包含：
- 功能描述
- 使用场景
- 预期效果
- 可能的实现方式（可选）

**Issue模板**：
```markdown
**功能描述**
我希望能够...

**使用场景**
当我...的时候，我需要...

**预期效果**
这个功能应该...

**可能的实现**
（可选）我认为可以通过...来实现
```

### 3. 改进文档

文档改进包括：
- 修正错别字
- 补充说明
- 添加示例
- 翻译文档

**如何贡献文档**：
1. Fork项目
2. 编辑文档文件
3. 提交Pull Request

### 4. 贡献代码

#### 代码规范

**Python代码**：
- 遵循PEP 8规范
- 使用类型提示
- 添加文档字符串
- 保持函数简洁（<50行）

**示例**：
```python
def parse_novel(file_path: str, character_name: str) -> Dict:
    """
    解析小说文本，提取指定角色的相关内容
    
    Args:
        file_path: 小说文件路径
        character_name: 角色名称
    
    Returns:
        包含对话、描写、场景的字典
    """
    # 实现...
    pass
```

**Prompt模板**：
- 使用清晰的markdown格式
- 包含详细的说明和示例
- 使用中文（如果是中文项目）
- 保持结构一致

#### 提交流程

1. **Fork项目**
```bash
# 在GitHub上点击Fork按钮
```

2. **克隆到本地**
```bash
git clone https://github.com/your-username/waifu-skill.git
cd waifu-skill
```

3. **创建分支**
```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

4. **进行修改**
```bash
# 编辑文件
# 测试修改
```

5. **提交更改**
```bash
git add .
git commit -m "feat: 添加XXX功能"
# 或
git commit -m "fix: 修复XXX问题"
```

**提交信息规范**：
- `feat:` 新功能
- `fix:` Bug修复
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 重构
- `test:` 测试相关
- `chore:` 构建/工具相关

6. **推送到GitHub**
```bash
git push origin feature/your-feature-name
```

7. **创建Pull Request**
- 在GitHub上点击"New Pull Request"
- 填写PR描述
- 等待审核

**PR模板**：
```markdown
**变更类型**
- [ ] Bug修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 代码重构
- [ ] 其他

**变更描述**
简要描述你的更改

**相关Issue**
Closes #123

**测试**
- [ ] 已测试
- [ ] 添加了测试用例

**检查清单**
- [ ] 代码遵循项目规范
- [ ] 更新了相关文档
- [ ] 所有测试通过
- [ ] 没有引入新的警告
```

## 贡献领域

### 优先级高的贡献

1. **Bug修复**
   - 修复已知Bug
   - 提高稳定性

2. **文档改进**
   - 补充缺失的文档
   - 添加更多示例
   - 翻译文档

3. **测试**
   - 添加单元测试
   - 添加集成测试
   - 测试边界情况

### 欢迎的贡献

1. **新的Isekai Bridge模板**
   - 为不同世界观创建模板
   - 优化现有模板

2. **工具改进**
   - 提高解析准确性
   - 支持更多格式
   - 优化性能

3. **Prompt优化**
   - 改进提取质量
   - 添加更多提示
   - 优化prompt结构

4. **示例角色**
   - 创建高质量示例
   - 覆盖不同类型
   - 提供完整原材料

### 未来功能（Phase 6）

如果你想贡献Phase 6的功能：
- 模板系统

## 代码审查

所有Pull Request都会经过审查：

**审查标准**：
1. 代码质量
2. 功能正确性
3. 文档完整性
4. 测试覆盖
5. 向后兼容性

**审查流程**：
1. 自动检查（如果有CI）
2. 人工审查
3. 讨论和修改
4. 批准和合并

## 行为准则

### 我们的承诺

为了营造开放和友好的环境，我们承诺：
- 尊重不同观点和经验
- 接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 我们的标准

**积极行为**：
- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

**不可接受的行为**：
- 使用性化的语言或图像
- 侮辱/贬损性评论和人身攻击
- 公开或私下骚扰
- 未经许可发布他人的私人信息
- 其他不道德或不专业的行为

### 执行

违反行为准则的行为将被处理：
1. 警告
2. 临时禁止
3. 永久禁止
