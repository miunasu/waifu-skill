# Waifu-Skill 更新日志

## v1.2.0 - 2024-04-08

### 🎯 重大改进：目标角色聚焦

#### 新增功能

**1. 目标角色提取强化**
- 在Lore Analyzer和Persona Analyzer中添加"⚠️ 重要原则：只提取目标角色的信息"部分
- 强调只提取目标角色的能力、经历、性格、台词
- 防止混入其他角色的信息

**2. 详细的错误示例**
- 提供常见混淆场景的错误示例和正确示例
- 特别针对双胞胎/相似角色（如雷姆和拉姆）
- 包含能力、经历、性格、台词等多个维度

**3. 提取检查清单**
- 为每条信息提供验证问题
- 确保提取的信息确实属于目标角色
- 要求明确的原文依据

## v1.1.0 - 2024-04-07

### 🎉 重大改进：关系定制系统

#### 新增功能

**1. 用户与角色关系定制**
- 在信息收集阶段，用户现在可以定义与角色的关系
- 不再需要用户定义角色性格（由LLM从原材料中提取）
- 支持5种预设关系类型：
  1. 成为原作中的重要人物
  2. 成为朋友
  3. 成为主人/召唤者
  4. 成为陌生人
  5. 自定义关系

**2. 智能冲突检测与解决**
- 自动检测用户要求与原作设定的冲突
- 提供4种解决方案：
  - 方案A：记忆重置
  - 方案B：身份替代
  - 方案C：情感转移
  - 方案D：保持原设定
- 在Isekai Bridge中实现解决方案

**3. 更合理的工作流程**
- 用户定义：与角色的关系
- LLM提取：角色的性格和行为
- 分工更清晰，结果更准确

#### 改进内容

**prompts/intake.md**
- ✅ 将"问题4：性格画像"改为"问题4：与角色的关系"
- ✅ 添加关系选项和示例
- ✅ 添加冲突检测和处理逻辑
- ✅ 更新输出格式，包含冲突解决信息

**docs/EXAMPLE.md**
- ✅ 更新示例对话，展示新的关系定制流程
- ✅ 添加冲突处理示例
- ✅ 更新"对用户的定位"部分，根据关系生成

**docs/WORKFLOW.md**
- ✅ 更新工作流程说明
- ✅ 强调关系定制的重要性

**docs/FAQ.md**
- ✅ 添加Q11：如何设定与角色的关系？
- ✅ 包含冲突处理说明

**docs/COMPLETION_SUMMARY.md**
- ✅ 更新功能描述
- ✅ 添加关系冲突检测功能

#### 使用示例

**场景1：成为原作中的重要人物**
```
用户: 我想成为昴君
结果: 雷姆会对你特别照顾和忠诚，就像对待原作中的昴君
```

**场景2：处理关系冲突**
```
用户: 我想成为雷姆的情侣，但我不是昴君
LLM: 检测到冲突，提供4种解决方案
用户: 选择方案A（记忆重置）
结果: 在Isekai Bridge中设定雷姆失去了对昴君的部分记忆
```

**场景3：自定义关系**
```
用户: 我想成为雷姆的哥哥
LLM: 好的，我会在Isekai Bridge中设定你是雷姆的哥哥
结果: 雷姆会把你当作哥哥，依赖和信任你
```

#### 技术细节

**数据结构变化**
```json
// 旧版本
{
  "personality": "温柔体贴 忠诚 有点自卑"
}

// 新版本
{
  "user_relationship": "我想成为昴君",
  "conflict_detected": false
}

// 有冲突时
{
  "user_relationship": "情侣",
  "conflict_detected": true,
  "conflict_type": "原作中角色已有恋人",
  "resolution_method": "方案A - 记忆重置",
  "resolution_details": "..."
}
```

**Isekai Bridge生成逻辑**
- 根据`user_relationship`生成"对用户的定位"部分
- 如果有冲突，根据`resolution_method`生成特殊说明
- 确保关系设定与角色性格一致

#### 优势

1. **更合理的分工**
   - 用户：定义关系（用户知道自己想要什么）
   - LLM：提取性格（LLM擅长从文本中提取信息）

2. **更灵活的定制**
   - 支持各种关系类型
   - 自动处理冲突
   - 保持角色一致性

3. **更好的用户体验**
   - 不需要用户理解角色性格
   - 提供清晰的选项
   - 智能解决冲突

4. **更准确的结果**
   - 性格由LLM从原作提取，更准确
   - 关系由用户定义，更符合需求
   - 冲突处理合理，不破坏人设

#### 向后兼容

- 旧版本的角色仍然可以使用
- 新创建的角色使用新的关系定制系统
- 可以通过更新Isekai Bridge来升级旧角色
