# Waifu-Skill 常见问题

## 基础问题

### Q1: Waifu-Skill是什么？

A: Waifu-Skill是一个角色蒸馏系统，能够从小说、设定集等原材料中提取虚拟角色的人格特征和世界观设定，生成可对话的AI Skill。核心创新是Isekai Bridge（异世界转换层），让角色能够合理地以AI形态存在。

### Q2: 需要什么技术背景？

A: 
- 基础：会使用命令行，会安装Python包
- 进阶：了解如何与LLM对话，理解prompt工程
- 不需要：编程经验、机器学习知识、GPU

### Q3: 支持哪些角色？

A: 理论上支持所有有文字原作的虚拟角色：
- ✅ 轻小说角色
- ✅ 动漫角色（有小说/漫画原作）
- ✅ 游戏角色（有剧情文本）
- ✅ 视觉小说角色
- ❌ 纯原创角色（没有足够原材料）
- ❌ 真人（伦理问题）

### Q4: 需要多少原材料？

A: 
- **最低要求**：角色出现100次以上的文本
- **推荐**：2-3卷小说，或等量的文本
- **最佳**：完整的原作 + 角色百科 + 设定集

### Q5: 生成一个角色需要多长时间？

A: 
- 预处理文本：5-10分钟
- LLM蒸馏：30-60分钟（取决于原材料量）
- 生成Skill：1-2分钟
- **总计**：约1-2小时

## 安装和配置

### Q6: 安装时提示"找不到模块"

A: 确保安装了所有依赖：
```bash
pip install -r requirements.txt
```

如果还是不行，尝试使用虚拟环境：
```bash
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### Q7: 无法解析EPUB/PDF文件

A: 
- EPUB：需要 `ebooklib` 和 `beautifulsoup4`
- PDF：需要 `PyPDF2`

```bash
pip install ebooklib beautifulsoup4 PyPDF2
```

### Q8: 在Windows上运行出错

A: 
- 使用 `python` 而不是 `python3`
- 路径使用反斜杠 `\` 或双反斜杠 `\\`
- 或者使用正斜杠 `/`（Python支持）

## 使用问题

### Q9: 如何选择世界观类型？

A: 根据角色所在的原作世界：
- **魔法世界**：有魔法、魔法师、魔法道具（如：Re:Zero、哈利波特）
- **科幻世界**：有高科技、太空、机器人（如：EVA、攻壳机动队）
- **修仙世界**：有修炼、灵气、法宝（如：凡人修仙传）
- **现代异能**：现代背景+超能力（如：某科学的超电磁炮）
- **古代历史**：古代背景，无魔法（如：三国、水浒）
- **游戏世界**：游戏设定（如：刀剑神域）

### Q10: 什么是Layer 0？为什么重要？

A: Layer 0是角色的核心人设（硬规则），包含：
- 身份认知：角色如何看待自己
- 核心信念：绝不放弃的原则
- 行为底线：绝对不会做的事
- 典型反应：固定的反应模式
- 语言禁忌：绝对不会说的话

**重要性**：Layer 0是最高优先级，任何情况下都不能违反。它确保角色的一致性和真实感。

### Q11: 如何设定与角色的关系？

A: 在信息收集阶段，你可以选择：
1. **成为原作中的重要人物**：如成为昴君（雷姆会对你特别照顾）
2. **成为朋友**：角色会友善但保持距离
3. **成为主人/召唤者**：角色会视你为召唤者
4. **成为陌生人**：角色会自然对待你
5. **自定义关系**：描述你想要的关系

**冲突处理**：如果你的要求与原作冲突（如：想成为情侣但角色已有恋人），LLM会提供解决方案：
- 方案A：记忆重置（角色失去部分记忆）
- 方案B：身份替代（你是平行世界的XX）
- 方案C：情感转移（角色逐渐对你产生好感）
- 方案D：保持原设定（维持友谊关系）

### Q12: 如何避免混淆相似角色的信息？

A: 这是v1.2.0重点解决的问题。在提取时：

**常见错误**：
- ❌ 把拉姆的风魔法当作雷姆的能力
- ❌ 把拉姆失去鬼角当作雷姆的经历
- ❌ 把拉姆的"拉姆是天才"当作雷姆的口头禅

**正确做法**：
1. **仔细确认原文中的角色名字**
2. **使用检查清单验证每条信息**：
   - 这是目标角色的能力吗？
   - 这是目标角色的经历吗？
   - 这是目标角色说的话吗？
3. **特别注意双胞胎/相似角色**
4. **每条信息都要有明确的原文依据**

**工具支持**：
- lore_analyzer.md和persona_analyzer.md已添加详细的错误示例
- 提供检查清单和验证问题
- 特别强调目标角色聚焦

### Q13: 如何判断提取质量？

A: 检查以下方面：
1. **有原作依据**：每条规则都能在原作中找到场景
2. **具体化**：不是"角色很温柔"，而是"在XX情况下会XX"
3. **包含反例**：不仅写会做什么，还写不会做什么
4. **前后一致**：没有矛盾的规则
5. **符合认知**：知识范围符合原作世界观
6. **目标角色聚焦**：没有混入其他角色的信息

### Q14: 角色回答不像原作怎么办？

A: 使用对话纠正功能：
1. 说"ta不会这样说"
2. LLM会识别纠正触发词
3. 解析纠正内容（场景、错误、正确）
4. 生成Correction记录
5. 更新文件并重新生成SKILL.md

示例：
```
用户: 你好
角色: 您好，很高兴见到您。
用户: 不对，雷姆不会这么正式，她应该说"昴君，雷姆一直在等您"
LLM: 我理解了，正在更新...
```

### Q15: 如何追加新的原材料？

A: 
1. 预处理新原材料
```bash
python3 tools/novel_parser.py \
  --file sources/new_volume.txt \
  --character "角色名" \
  --output parsed/new_volume_dialogues.txt \
  --extract-dialogues \
  --context-lines 2
```

2. 让LLM读取 `prompts/merger.md`
3. LLM分析增量内容，检测冲突
4. 用户确认合并建议
5. 备份当前版本
6. 更新文件并重新生成SKILL.md

### Q14: 版本管理如何使用？

A: 
```bash
# 备份当前版本
python3 tools/version_manager.py --action backup --slug rem

# 列出历史版本
python3 tools/version_manager.py --action list --slug rem

# 回滚到指定版本
python3 tools/version_manager.py --action rollback --slug rem --version v2

# 比较版本差异
python3 tools/version_manager.py --action compare --slug rem --version1 v1 --version2 v2
```

## 高级问题

### Q15: 如何处理多个版本的同一角色？

A: 为不同版本创建不同的slug：
- `rem-arc3` - 第三章的雷姆
- `rem-if` - IF线的雷姆
- `misaka-railgun` - 超电磁炮版的美琴
- `misaka-index` - 魔禁版的美琴

### Q16: 如何创建自定义Isekai Bridge模板？

A: 
1. 在 `templates/` 目录创建新模板
2. 参考 `prompts/isekai_bridge_generator.md` 的格式
3. 定义转换机制、术语映射、能力限制
4. 在生成时指定使用自定义模板

### Q17: 原材料不足怎么办？

A: 可以补充：
- 角色百科（Wiki）
- 官方设定集
- 动画脚本
- 漫画对话
- 用户自己的描述（谨慎使用）
- 同人作品（非常谨慎）

**注意**：补充材料要与原作一致，不要引入矛盾信息。

### Q18: 如何测试角色质量？

A: 使用以下测试方法：

**1. 原作场景重现测试**
```
用户: （描述原作中的场景）
角色: （应该给出符合原作的反应）
```

**2. 边界情况测试**
```
用户: 你比你姐姐强吧？
雷姆: 不、不是的！雷姆怎么可能比姐姐大人强呢？
（✅ 正确：触发了Layer 0的行为底线）
```

**3. 知识范围测试**
```
用户: 你知道电脑是什么吗？
雷姆: 电脑？那是...这个世界的魔法道具吗？
（✅ 正确：不知道现代科技）
```

**4. 一致性测试**
```
多次对话，检查角色是否前后一致
```

### Q19: 如何优化角色表现？

A: 
1. **增加原材料**：更多文本 = 更准确的提取
2. **细化Layer 0**：添加更多具体的硬规则
3. **补充示例对话**：在Layer 2中添加更多示例
4. **使用对话纠正**：根据实际对话不断优化
5. **定期回顾**：检查Correction记录，合并相似项

### Q20: 可以创建多角色对话吗？

A: 
目前版本支持单角色对话。多角色对话是Phase 6的高级功能，需要：
- 角色关系网络
- 多角色对话模式
- 角色互动规则

可以通过创建多个角色，然后手动切换来模拟多角色对话。

## 故障排除

### Q21: 生成的SKILL.md格式不对

A: 检查：
1. `lore.md`, `persona.md`, `isekai_bridge.md` 格式是否正确
2. 是否有特殊字符导致解析错误
3. 文件编码是否为UTF-8
4. 是否有未闭合的markdown标记

### Q22: 角色行为不稳定

A: 可能原因：
1. **Layer 0不够具体**：添加更多硬规则
2. **有矛盾的规则**：检查并解决冲突
3. **Correction记录过多**：合并相似项
4. **原材料有冲突**：检查原材料质量

### Q23: 工具提示"文件不存在"

A: 
1. 检查文件路径是否正确
2. 使用绝对路径或相对路径
3. 确保文件名和扩展名正确
4. 检查文件权限

### Q24: LLM提取的内容不准确

A: 
1. **检查prompt**：确保LLM读取了正确的prompt
2. **增加原材料**：更多文本 = 更准确
3. **明确指示**：在prompt中强调"必须有原作依据"
4. **人工审核**：检查提取结果，手动修正

## 伦理和法律

### Q25: 使用Waifu-Skill是否合法？

A: 
- ✅ 个人学习和娱乐使用：合法
- ✅ 非商业用途：合法
- ❌ 商业用途：需要版权方授权
- ❌ 生成有害内容：违法

**建议**：仅用于个人娱乐，尊重原作版权。

### Q26: 可以分享创建的角色吗？

A: 
- ✅ 分享创建方法和技巧：可以
- ✅ 分享prompt模板：可以
- ⚠️ 分享生成的SKILL.md：谨慎（可能涉及版权）
- ❌ 商业销售：不可以

**建议**：分享时注明原作出处，不用于商业用途。

### Q27: 如何避免生成有害内容？

A: 
1. 选择正面的角色
2. 在Layer 0中添加行为底线
3. 定期检查对话内容
4. 使用对话纠正功能
5. 遵守平台规则
