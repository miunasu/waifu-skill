# Intake - 信息收集

> 通过对话收集角色基础信息

## 收集流程

### 1. 角色名称（必填）

**提示**：
```
这个角色叫什么名字？
（例如：雷姆 / 炮姐 / 阿库娅）
```

**存储**：`name`

---

### 2. 作品信息（可选）

**提示**：
```
角色来自哪部作品？
（例如：Re:从零开始的异世界生活）
（可跳过）
```

**存储**：`work`

---

### 3. 与角色的关系（重要，必填）

**提示**：
```
你希望与角色是什么关系？

1. 成为角色在原作中喜欢/重视的人
2. 成为角色的朋友
3. 成为角色的主人/召唤者
4. 陌生人，让角色自然对待
5. 自定义关系（请描述）
```

**存储**：`user_relationship`

**验证**：
- 必填项，不能跳过
- 如果用户未提供，必须询问

**注意**：如果与原作设定冲突，将在Isekai Bridge生成时处理。

---

### 4. 世界观类型（跳过）

**提示**：
```
角色所在的世界类型？（将从原材料中分析）

1. 魔法世界
2. 科幻世界
3. 修仙世界
4. 现代异能
5. 古代历史
6. 游戏世界
7. 其他（请描述）
8. 跳过
```

**存储**：`world_type`

---

### 汇总展示

**提示**：
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
角色名称：{name}
作品来源：{work 或 "未提供"}
与角色的关系：{user_relationship}
世界观类型：{world_type 或 "将从原材料中分析"}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

注：性别、年龄、身份、种族、世界观类型等将从原材料中自动提取。

```

**重要**：
- "与角色的关系"是必填项，必须在Step 1收集
- 如果用户未提供，必须询问
- 不能标注为"将从原材料中分析"

---

## 输出格式

```json
{
  "name": "雷姆",
  "slug": "rem",
  "work": "Re:从零开始的异世界生活",
  "user_relationship": "我想成为昴君",
  "world_type": "魔法世界",
  "created_at": "2024-04-07T12:00:00Z"
}
```

**slug生成规则**：
- 转小写
- 空格→连字符
- 移除特殊字符
- 仅保留字母、数字、连字符
