# Merger - 增量原材料合并

> 当用户追加新的原材料时，分析增量内容，检测冲突，更新文件

## 核心原则

1. **不覆盖已有信息**：追加不覆盖，补充不替换
2. **保持一致性**：新旧信息要协调，不能出现矛盾
3. **版本管理**：合并前自动备份，支持回滚

---

## 工作流程

### 1. 读取新原材料

**输入**：新的小说章节、补充设定文档、官方设定集

**处理**：
1. 使用novel_parser.py预处理文本
2. 提取新的角色信息
3. 标记为"增量内容"

---

### 2. 分析增量内容

使用`lore.md`和`persona.md`提取新信息。

---

### 3. 冲突检测

**Lore冲突示例**：
```markdown
### 冲突：角色年龄
- 现有：17岁
- 新增：18岁
- 建议：可能是时间线推进，建议更新
```

**Persona冲突示例**：
```markdown
### 冲突：性格表现
- 现有：从不生气
- 新增：在某场景中大发雷霆
- 建议：可能是特殊情况，建议在Layer 0添加例外规则
```

---

### 4. 用户确认

```markdown
## 合并预览

### 将要进行的更新
- Lore - 角色背景：追加事件"白鲸战"
- Persona - Layer 2：新增口头禅"雷姆会保护昴君"

### 检测到的冲突
- 冲突1：{description}

是否继续合并？
1. 确认合并
2. 取消操作
```

---

### 5. 执行合并

1. 备份当前版本
2. 更新lore.md和persona.md
3. 重新生成SKILL.md
4. 更新meta.json

---

## 合并规则

### Lore合并规则

| 维度 | 合并策略 |
|------|---------|
| 世界观 | 补充细节，不覆盖基本设定 |
| 角色背景 | 追加事件，按时间排序 |
| 能力体系 | 追加技能，标注获得时间 |
| 知识范围 | 扩展列表，只增不减 |
| 价值观 | 补充细节，核心信念不变 |
| 经典场景 | 追加场景 |

### Persona合并规则

| Layer | 合并策略 |
|-------|---------|
| Layer 0 | 通常不变，除非用户明确纠正 |
| Layer 1 | 补充信息，基本身份不变 |
| Layer 2 | 追加口头禅和示例 |
| Layer 3 | 补充情感表达方式 |
| Layer 4 | 追加互动场景 |
